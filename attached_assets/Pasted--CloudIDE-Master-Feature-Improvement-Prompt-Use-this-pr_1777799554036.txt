# CloudIDE — Master Feature & Improvement Prompt

Use this prompt (or sections of it) when working with an AI to build new features or improve existing ones. The codebase is a full-stack cloud IDE: React + TypeScript frontend (wouter routing, CodeMirror editor, shadcn/ui, Tailwind), Node.js/Express backend, BullMQ + Redis job queues, PostgreSQL via Drizzle ORM, and Flutter/Android build pipelines.

---

## 🧠 CONTEXT BLOCK (paste at top of every session)

```
You are working on CloudIDE — a browser-based code editor and execution platform.

Stack:
- Frontend: React + TypeScript, wouter routing, CodeMirror 6, shadcn/ui, Tailwind CSS
- Backend: Node.js + Express, BullMQ workers, Redis, PostgreSQL (Drizzle ORM)
- Build pipeline: Flutter SDK + Android SDK on server, APK streaming via BullMQ
- Auth: JWT in httpOnly cookies, optional guest mode via user-key
- Execution: sandboxed server-side runners (Python, JS, Dart, etc.) with SSE streaming
- Sharing: public share links with fork/run event tracking, Explore gallery
- File storage: project files stored in PostgreSQL (versions table)

Key existing hooks: useRun, useBuild, useProjects, useAuth, useSnackSync
Key existing components: Editor, FileTree, TabBar, PreviewPanel, MobilePreview, ShareModal, CommandPalette, BuildLog, ConsoleOutput, Toolbar, StatusBar
Key existing routes: /auth, /projects, /run, /build, /share, /metrics
```

## 🏗️ ARCHITECTURE IMPROVEMENTS

```
Technical debt and infrastructure improvements:

1. WEBSOCKET LAYER — Replace all SSE streaming (run, build) with a unified WebSocket 
   manager. Single /ws connection, multiplexed channels by type. Cleaner reconnect logic.

2. FILE STORAGE MIGRATION — Move project files out of PostgreSQL JSON blobs into 
   object storage (S3/R2/MinIO). Store only metadata + file refs in DB. 
   Enables: larger projects, binary files, CDN delivery.

3. TYPED API CLIENT — Generate a typed API client from Express route schemas using 
   zod-to-openapi + openapi-typescript. Replace all raw fetch() calls in hooks.

4. ERROR BOUNDARY — Add React ErrorBoundary around Editor, PreviewPanel, MobilePreview.
   Show graceful fallback instead of white screen on component crash.

5. OPTIMISTIC UPDATES — useProjects: make save/rename/delete optimistic (update UI 
   immediately, rollback on error). Currently likely has loading delays.

6. REDIS CACHING — Cache GET /api/share/:id responses in Redis (TTL 60s). 
   Share pages are public and hit DB on every view.

7. BUILD QUEUE PRIORITY — Add priority levels to BullMQ build queue: 
   Pro users get priority: 1, free users get priority: 10. 
   Implement in buildJobProcessor.ts using job.opts.priority.

8. STRUCTURED LOGGING — Replace console.log with the existing Pino logger (buildStreams) 
   consistently across all route handlers and workers.

9. HEALTH CHECK ENDPOINT — GET /health returns: DB status, Redis status, queue depths, 
   Flutter/Android SDK availability. Use in deployment monitoring.

10. RATE LIMIT HEADERS — Return X-RateLimit-Remaining, X-RateLimit-Reset headers 
    from the rate limiter middleware. Frontend can show quota UI without extra API call.
```
