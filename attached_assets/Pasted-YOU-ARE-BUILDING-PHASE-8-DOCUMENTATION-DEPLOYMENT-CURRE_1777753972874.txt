     ```
     OPTIONAL:
     - FLUTTER_ENABLED=true/false (auto-detect)

     - GITHUB_TOKEN (for Android builds via GitHub)

     - DAILY_BUILD_LIMIT=3
     ```

3. OPERATIONAL RUNBOOK
   - Startup checklist:
     * Verify Flutter (if enabled)
     * Verify Redis connection
     * Verify PostgreSQL connection
     * Check disk space
     * Migrate database schema
   
   - Troubleshooting:
     * "Build queue stuck": Check Redis, restart queue processor
     * "APK not downloading": Check disk space, verify apk_path in DB
     * "Job timeout": Check server load, increase timeout in env var
     * "Auth failing": Verify JWT_SECRET matches across servers
   
   - Scaling guide:
     * Multiple queue workers: Run processor on separate node (scale horizontally)
     * Multiple API servers: Load balance, share DB + Redis
     * Database: Use managed PostgreSQL (Neon, AWS RDS)
     * Redis: Use managed Redis (AWS ElastiCache, Upstash)
     * APK storage: Switch to S3 for distributed storage
     * CDN: Optional, for shared project preview
